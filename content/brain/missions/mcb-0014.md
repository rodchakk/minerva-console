## Summary

Created the minimal, Git-backed, Markdown-first coordination skeleton under `content/brain/loop/` so AI agents (GPT, Claude, Codex, Gemini) can hand off work through the repo instead of through Rudy. Folder = status: a mission brief's state is the folder it lives in (`01_todo` … `05_blocked`), with no `QUEUE.json` by design. Skeleton only — no scripts, GitHub Actions, scheduler, bot, or agent engine.

## Scope

- Add `content/brain/loop/**`: `PROTOCOL.md`, `ROLES.md`, `OPERATOR_GUIDE.md`, mission status folders, per-agent report folders, and brief/report templates.
- Document the evidence discipline (verified / inferred / unknown), one-writer-per-branch, Rudy as sole merge owner, and that GitHub/Git are authority for branch/diff/CI/merge while Markdown is auditable handoff.
- Add the Delegated Git Ops rules so agents can run stage/commit/push/PR with merge gated by explicit approval.
- Keep it Git-backed and read-only — no Neon, Supabase, ENTRY, Seshat, RAG, embeddings, agent engine, or UI write path.
